Valine评论系统样式定制指南

本指南帮助你定制Valine评论系统的样式，包括原生评论系统和加密评论功能。

文件说明

  • valine-custom.styl: 包含所有Valine样式的自定义文件
  • 原始Valine: Valine的原始JS和样式已下载到本地，位于/themes/amazing/source/js/valine/目录

样式定制方法

基本步骤

  1. 打开themes/amazing/source/css/valine-custom.styl文件
  2. 根据需要修改相应的CSS样式
  3. 重新生成站点： hexo clean && hexo generate
  4. 查看效果并调整

主要样式类别

Valine使用的主要CSS类别如下：

评论容器

.v - 整个评论区的容器

评论表单

  • .vwrap - 评论输入表单容器
  • .veditor - 评论内容输入框
  • .vinput - 表单输入框（昵称、邮箱等）
  • .vsubmit - 提交按钮

评论列表

  • .vlist - 评论列表容器
  • .vcard - 单个评论容器
  • .vhead - 评论头部（用户信息）
  • .vnick - 评论者昵称
  • .vcontent - 评论内容
  • .vmeta - 评论元数据（时间、回复按钮等）

加密评论相关

  • .vencrypt-container - 加密选项容器
  • .vencrypt-toggle - 加密开关
  • .vencrypt-password - 加密密码输入框
  • .vencrypted-comment - 加密评论容器
  • .vencrypted-tip - 加密评论提示
  • .vencrypted-form - 解密表单
  • .vencrypted-password - 解密密码输入框
  • .vencrypted-submit - 解密按钮
  • .vdecrypted-content - 解密后的内容容器

样式覆盖技巧

  1. 使用更高的CSS优先级来覆盖原始样式
  2. 使用!important关键字在必要时强制应用样式
  3. 利用Stylus的嵌套功能组织样式，提高可维护性

色彩与尺寸定制

可以创建变量来统一管理颜色和尺寸：

// 在valine-custom.styl文件顶部添加变量
$primary-color = #1abc9c
$border-radius = 8px
$font-family = 'Ubuntu', 'Microsoft YaHei', sans-serif

// 在样式中使用变量
.vsubmit {
  background-color: $primary-color
  border-radius: $border-radius
}

示例：自定义按钮样式

// 修改提交按钮样式
.v .vwrap .vsubmit {
  background-color: #ff6b6b
  color: white
  padding: 8px 18px
  border-radius: 20px
  transition: all 0.3s ease
  
  &:hover {
    background-color: #ff5252
    transform: translateY(-2px)
    box-shadow: 0 4px 8px rgba(0,0,0,0.1)
  }
}

注意事项

  1. Valine的样式被加载在其JS文件中，我们的自定义样式需要具有更高的优先级
  2. 每次修改后请清理缓存再生成站点
  3. 在开发者工具中查看样式，有助于定位需要修改的CSS类
  4. 避免直接修改Valine原始JS文件，而是通过CSS覆盖来实现样式定制
